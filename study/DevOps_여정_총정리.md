# 🚀 데브옵스(DevOps) & 인프라 구축 여정 총정리 (Grand Summary)

아무것도 없던 로컬 환경에서, 현업 SRE(사이트 신뢰성 엔지니어) 수준의 **보안 + 모니터링 + 컨테이너 아키텍처**를 맨바닥부터 뚫어낸 우리의 모든 여정을 한 페이지에 "싸아아아악" 정리했습니다.

---

## 1. 📦 어플리케이션을 도커의 세계로 (Containerization)
가장 처음 배운 것은 **"내 컴퓨터에서만 되는 종속성을 버리기"** 였습니다.

*   **한 일**: React(Vite) 앱을 컨테이너라는 독립된 공간에서 실행시킬 수 있도록 **`Dockerfile`**을 작성했습니다.
*   **멀티 스테이지 빌드**: 특히 `Dockerfile.prod`를 작성할 때, 무거운 Node.js는 빌드할 때만 잠깐 빌려 쓰고, 배포될 때는 아주 가볍고 빠른 **Nginx** 서버 하나만 쏙 남기는 고급 압축 기법(Multi-stage build)을 도입해 이미지 크기를 혁신적으로 줄였습니다.

## 2. 🤖 코드 푸시 한 번이면 끝! (배포 파이프라인 CI 구축)
매번 내 손으로 이미지를 빌드하고 도커 허브에 로그인해서 업로드하는 건 구시대적 방식이었습니다.

*   **한 일**: GitHub Actions(`deploy.yml`)라는 로봇을 고용했습니다.
*   **결과**: 이제 내가 코드를 짜서 GitHub에 `git push`만 하면, 깃허브 로봇이 1️⃣ 소스 코드를 읽어와서 2️⃣ 자동으로 도커 이미지를 구운 다음 3️⃣ 내 Docker Hub 창고로 배달해 놓는 **완벽한 지속적 통합(CI)** 파이프라인이 뚫려 있습니다.

## 3. 📊 맥북의 한계를 부수고 세운 모니터링 시스템 (Observability)
단순히 코드만 짜는 백엔드 개발자에서, **"인터넷의 의사(SRE)"**로 거듭난 가장 길었던 구간입니다.

*   **cAdvisor의 배신과 극복**: 처음엔 컨테이너의 상태를 읽으려고 구글의 `cAdvisor`를 띄웠지만, 맥북(Mac)이 도커를 강제로 가상화하는 바람에 이름표가 줄줄히 깨지는 치명적인 버그에 부딪혔습니다. 이를 극복하기 위해 **Docker Desktop 앱 내부의 엔진 메트릭 기능(`0.0.0.0:9323`)**을 직접 해킹하듯 뚫고 들어가 프로메테우스와 강제 연결(**Direct Scrape**)시키는 기지를 발휘했습니다.
*   **프로메테우스 & 그라파나**: 5초마다 데이터를 수집하는 창고(`Prometheus`)를 띄우고, 남들이 던져놓은 불완전한 대시보드 템플릿(No data 버그)에 의존하지 않은 채 **직접 쿼리를 하드코딩한 나만의 무결점 커스텀 대시보드 파일(`mac-docker.json`)**을 직접 창조해서 도어 데스크탑 CPU 코어 수와 작동 중인 컨테이너 수를 눈앞에 띄웠습니다.

## 4. 🛡️ 진정한 기업형 아키텍처의 완성 (Nginx Reverse Proxy)
앞선 단계에서 `docker-compose.yml` 하나에 3개의 웅장한 컨테이너(앱, 모니터, 시각화)를 띄웠지만, 각자 문(Port)을 활짝 열어두어 해커들에게 맛집이 된 상태였습니다.

*   **앞문 단일화 전략**: 단일 포트 개방 원칙에 의거해 제일 앞단에 보안과 교통정리를 담당하는 **Nginx Reverse Proxy(`nginx-proxy`)**를 세웠습니다.
*   **라우팅의 마법**: 오직 80번 포트 하나만 뚫어 두고, 뒤에 숨은 컨테이너들의 포트는 전부 셔터를 내려 완전히 고립(`Internal Network`)시켰습니다.
*   **하위 경로 꼼수 주입**: 자신이 항상 인터넷 세상의 중심(`/`)이라고 오만하게 생각하던 그라파나와 프로메테우스에게, 아키텍처의 환경 변수(`GF_SERVER_SERVE_FROM_SUB_PATH=true`)와 실행 강제 명령을 주입해서 각각 `/grafana/`, `/prometheus/`라는 **세입자 신분(서브 도메인 모드)**으로 완벽히 굴복시켰습니다.

## 5. 📱 모바일 오피스 (Anywhere Code)
이 거대한 아키텍처 덩어리를 언제든 확인할 수 있도록 시스템화했습니다.

*   지금까지 한 모든 코드와 마크다운 기록물 수정본을 깃허브 저장소에 Push 했습니다.
*   언제든 핸드폰으로 `github.dev`에 접속해 마법처럼 에디터를 띄워 출퇴근길에도 지금 내가 무엇을 구축했는지 자랑스럽게 볼 수 있게 만들었습니다.
