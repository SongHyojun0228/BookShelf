## 알아야 할 것! : **컴포넌트 실행 순서**
### 1. [최초 렌더링 시작] : bookDetail 컴포넌트 호출
### 2. useState가 초기화
-  bookData는 명시하신 대로 **null**.
### 3. [화면 그리기] return (...) 안의 UI를 브라우저에 먼저 그리려고 시도
- 이때 bookData.title을 만나는데, 현재 bookData는 null이므로, null.title을 읽으려다가 에러 화면(스크린다운) 발생
### 4. [useEffect 실행] (에러가 나지 않고 그리기 끝났을 경우에만) 
- 화면을 다 그린 직후에 비로소 useEffect가 실행되어 서버에 API 요청 

즉, useEffect 안에서 데이터를 잘 걸렀다해도 서버에 데이터를 요청하기도 전인 찰나의 순산(최초 렌더링)에 화면부터 먼저 그리려고 시도하기 때문에 TS가 null일 수도 있는 변수를 캐치.

## 그래서 어떻게 해결한 건데?
## ?(옵셔널 체이닝) 사용
`{bookData?.title}`

## ?를 사용하기 싫다면, 화면에 그리기 전에 로딩처리를 해서 아예 밑으로 내려가지 못 하게 막기
- if문으로 렌더링 막기 
    - return (...)으로 화면을 다 그리기 전에, bookData가 없으면 로딩 화면만 보여주도록.
