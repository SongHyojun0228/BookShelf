***[ 이미지 추가 기능 구현 가이드 ]***

## 1. Supabase Storage 세팅 (사전 작업)
- **Supabase Storage Bucket 생성**
- **이름**: `review_images`
- **권한**: `Public` (누구나 이미지를 조회할 수 있도록 설정)
- **DB 컬럼 추가**: `Reviews` 테이블에 `image_urls` (Type: `text[]` 배열) 컬럼 생성

## 2. 상태 관리 (State) 추가
- `selectedFiles`와 미리보기용 `previewUrls`를 추가
```ts
const [selectedFiles, setSelectedFiles] = useState<File[]>([]);
const [previewUrls, setPreviewUrls] = useState<string[]>([]);
```

## 3. 이미지 선택 및 미리보기 로직
- **handleImageSelect**: 파일이 선택되면 최대 3장까지 제한을 체크하고, `URL.createObjectURL()`을 생성하여 화면에 즉시 렌더링.
- **handleRemoveImage**: 미리보기 우측 상단 `✕` 버튼 클릭 시 해당 인덱스의 파일과 URL을 상태에서 제거.

## 4. Supabase Storage 업로드 및 DB 저장
`handleAddReview` 함수 실행 시 다음 순서로 처리.
1. **파일 업로드**: `selectedFiles`에 담긴 파일들을 `review_images` 버킷에 업로드
   - 파일명 중복을 막기 위해 `${Date.now()}_` 접두어 추가
2. **URL 획득**: 업로드 성공 시 `getPublicUrl`을 통해 공개 이미지 링크를 가져옴.
3. **DB 저장**: 최종적으로 얻은 URL 배열을 `Reviews` 테이블의 `image_urls` 컬럼에 넣어 저장
---
## 💡 팁:
**파일명 변환 (중요!)**: 파일명에 한글이나 공백이 있으면 Supabase Storage에서 `Invalid key` 에러가 날 수 있습니다. 업로드 시 아래와 같이 안전한 형식으로 변환하여 저장하는 것이 좋습니다.
```ts
const fileExt = file.name.split('.').pop(); // 확장자 추출
const fileName = `${Date.now()}_${Math.floor(Math.random() * 1000)}.${fileExt}`;
```
