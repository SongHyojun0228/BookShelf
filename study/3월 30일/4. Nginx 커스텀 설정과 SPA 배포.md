# 4. Nginx 커스텀 설정과 SPA(React) 배포

리액트(React)와 같은 Single Page Application(SPA)을 Nginx와 도커로 배포할 때 맞닥뜨리는 두 가지 실전 과제를 해결하는 방법

---

## 1. 새로고침 시 404 에러 문제 (The 404 Problem)
### 원인
- 리액트 라우터(`react-router-dom`)는 주소창의 경로(`/books/1` 등)를 자바스크립트로 처리하는 가짜 경로
- 하지만! 사용자가 **새로고침**을 하면, Nginx 서버는 실제로 서버 안에 `/books/1`이라는 폴더나 파일이 있는지 찾기 시작
- 당연히 서버에는 그런 파일이 없으므로 Nginx는 당당하게 **404 Not Found**를 띄움

### 해결책: `try_files`
Nginx 설정에 다음 한 줄을 추가합니다.
```nginx
try_files $uri $uri/ /index.html;
```
- **해석**: "먼저 사용자가 요청한 주소($uri)가 파일로 있는지 보고, 없으면 폴더인지 봐. 그것도 없으면? **그냥 `/index.html`을 보여줘!** 나머지는 리액트 라우터가 알아서 할 거야~"

---

## 2. API 프록시 설정 (Gateway 역할)
### 문제
- 리액트 개발 시에는 Vite의 프록시 설정으로 알라딘 API와 통신했습니다.
- 하지만~ 실전 배포(Nginx)에서는 Vite 서버가 없으므로 알라딘 API로 직접 요청을 보낼 때 **CORS 에러**가 발생할 수 있습니다.

### 해결책: Nginx Proxy Pass
Nginx가 중간에서 요청을 가로채 알라딘 서버로 전달하도록.
```nginx
location /aladin-api/ {
    proxy_pass http://www.aladin.co.kr/;
    proxy_set_header Host www.aladin.co.kr;
}
```

---

## 3. 실전 적용: nginx.conf 작성법
프로젝트 루트에 `nginx.conf` 생성 

```nginx
server {
    listen 80;

    location / {
        root   /usr/share/nginx/html;
        index  index.html;
        try_files $uri $uri/ /index.html; # 새로고침 해결
    }

    # API 프록시 (Vite 설정과 동일하게 맞춤)
    location /aladin-api/ {
        proxy_pass http://www.aladin.co.kr/;
        proxy_set_header Host www.aladin.co.kr;
    }
}
```

## 4. Dockerfile.prod 업데이트
Nginx가 기본 설정을 버리고 **우리가 만든 `nginx.conf`를 사용하도록** 한 줄을 추가

```dockerfile
# ... (앞부분 생략) ...
FROM nginx:stable-alpine

# 커스텀 설정 복사 (경로 주의!)
COPY nginx.conf /etc/nginx/conf.d/default.conf

# 빌드물 복사
COPY --from=build /app/dist /usr/share/nginx/html

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```
