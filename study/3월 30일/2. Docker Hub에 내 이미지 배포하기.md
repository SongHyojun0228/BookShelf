# 2. Docker Hub에 내 이미지 배포하기 (실습 가이드)

지금까지 만든 `bookshelf-prod` 이미지는 당신의 컴퓨터(로컬)에만 존재합니다. 이를 **Docker Hub**라는 온라인 저장소에 올려서, 세상 어디서든 이 이미지를 가져올 수 있게 만들어 봅시다.

---

### Step 1: Docker Hub 로그인 (CLI)
터미널에서 도커 허브 계정에 접속합니다.

```bash
docker login
```
- 아이디(Username)와 비밀번호를 차례로 입력합니다. (비밀번호 입력 시 화면에 아무것도 안 나오지만, 잘 입력되고 있는 것이니 당당하게 치면 됩니다!)
- **결과**: `Login Succeeded`가 나오면 성공!

```bash
hyojun@songhyojun-ui-MacBookAir ~ % docker login

Authenticating with existing credentials... [Username: thdgywns2300]
i Info → To login with a different account, run 'docker logout' followed by 'docker login'
Login Succeeded
```

---

### Step 2: 내 이미지에 이름표 붙이기 (Tagging)
도커 허브는 `내아이디/이미지이름` 형식만 업로드를 허용합니다. 현재 로컬에 있는 이미지를 이 형식으로 복사(이름 붙이기)합니다.

*   **본인의 Docker Hub ID를 아래 `[내아이디]` 칸에 적어주세요.**
```bash
# 예시: docker tag bookshelf-prod:latest hyojun/bookshelf-prod:latest
docker tag bookshelf-prod:latest [내아이디]/bookshelf-prod:latest
```

---

### Step 3: 이미지 업로드 (Push)
이제 진짜로 인터넷(Docker Hub) 서버로 이미지를 밀어 넣습니다.

```bash
docker push [내아이디]/bookshelf-prod:latest
```
- 데이터가 업로드되는 바가 쭉 올라갈 것입니다.
- 완료되면 인터넷 브라우저로 [Docker Hub](https://hub.docker.com/)에 접속해서 내 **Repositories**에 이미지가 올라왔는지 확인해 보세요!

```bash
hyojun@songhyojun-ui-MacBookAir ~ % 
docker tag bookshelf-prod:latest thdgywns2300/bookshelf-prod:v1.0.1
```

---

### Step 4: 진짜 'DevOps'의 맛 느끼기 (Pull & Run)
이제 당신의 이미지는 구름 속에 있습니다. 기존에 내 컴퓨터에 있던 이미지들을 다 지우고, 도커 허브에서만 다시 받아와서 실행해 봅시다. (마치 다른 사람인 것처럼!)

```bash
# 1. 내 컴퓨터에 있던 기존 컨테이너 중지 및 삭제
docker stop bookshelf-prod-test
docker rm bookshelf-prod-test

# 2. 내 컴퓨터에 있던 이미지 삭제 (깨끗한 상태)
docker rmi [내아이디]/bookshelf-prod:latest

# 3. 도커 허브에서 다시 받아와서 실행하기
docker run -d -p 8080:80 --name bookshelf-cloud-test [내아이디]/bookshelf-prod:latest
```
*   **결과**: 소스코드가 하나도 없어도, 이미지만으로 웹사이트가 척척 실행되는 마법을 확인하세요! [http://localhost:8080](http://localhost:8080)

---

### Step 5: 업데이트 및 버전 관리 (Update & Versioning)

이미 업로드한 이미지를 수정해서 다시 올리고 싶다면(재배포), 어떻게 해야 할까요?

#### 1) 기존 이미지 덮어쓰기 (`latest`)
가장 단순한 방법으로, 똑같은 이름으로 다시 빌드하고 Push하면 도커 허브의 이미지가 최신 버전으로 바뀝니다.
```bash
# 코드 수정 후 다시 빌드
docker build -f Dockerfile.prod -t bookshelf-prod .
# 같은 태그로 이름표 붙이기
docker tag bookshelf-prod:latest [내아이디]/bookshelf-prod:latest
# 다시 Push (바뀐 부분만 올라갑니다!)
docker push [내아이디]/bookshelf-prod:latest
```

#### 2) 버전별로 관리하기 (추천!)
실무에서는 서비스의 안정성을 위해 버전을 명시합니다. (예: `v1.0`, `v1.1`)
```bash
# 태그 이름에 버전을 명시
docker tag bookshelf-prod:latest [내아이디]/bookshelf-prod:v1.0.1
# 해당 버전으로 Push
docker push [내아이디]/bookshelf-prod:v1.0.1
```
- **장점**: 만약 새로 올린 버전(v1.0.1)에 버그가 있다면, 즉시 이전 버전(v1.0.0)으로 **롤백(Rollback)**하기 매우 쉽습니다.

#### 💡 Docker의 마법: "이미 올라간 건 안 올려요"
실제로 다시 `push`해 보면, `Layer already exists`라는 메시지가 나오며 수초 만에 끝나는 것을 볼 수 있습니다. 도커는 똑똑하게도 **파일이 바뀐 레이어(Layer)만** 다시 전송하기 때문입니다! 효율적인 배포의 핵심입니다.
