# 3. GitHub Actions로 자동 빌드 및 업로드 (CI 자동화)
- 지금까지 우리는 내 컴퓨터(로컬)에서 직접 명령어를 쳐서 도커 이미지를 만들고 업로드
- 하지만! 이제는 **GitHub**이 이 일을 대신 하도록 시킬 것!
- 이것이 바로 **CI (Continuous Integration, 지속적 통합)**의 핵심

---

### Step 1: GitHub Secrets 설정 (보안)
GitHub 로봇이 내 대신 Docker Hub에 로그인하려면 비밀번호가 필요 
하지만 소스 코드에 비밀번호를 적어둘 수 없기 때문에 **GitHub Secrets** 사용

1.  GitHub 웹사이트의 내 프로젝트 저장소(Repository)로 이동
2.  **Settings** > **Secrets and variables** > **Actions**
3.  **New repository secret** 버튼을 눌러 아래 변수들을 등록
    - `DOCKER_USERNAME`: 본인의 Docker Hub ID
    - `DOCKER_PASSWORD`: 본인의 Docker Hub 비밀번호 (또는 Access Token)
    - `VITE_SUPABASE_URL`: 프로젝트의 Supabase URL
    - `VITE_SUPABASE_ANON_KEY`: 프로젝트의 Supabase Key

---

### Step 2: 자동화 업무 지시서 작성 (Workflow)
프로젝트 폴더에 `.github/workflows` 폴더를 만들고, `deploy.yml` 파일을 생성 (이 파일이 바로 로봇이 읽을 주문서)

**파일 경로**: `.github/workflows/deploy.yml`

```yaml
name: Docker Build & Push

on:
  push:
    branches: [ main ] # main 브랜치에 푸시될 때마다 실행

jobs:
  build-and-push:
    runs-on: ubuntu-latest # GitHub이 제공하는 가상 컴퓨터(Ubuntu)에서 실행
    
    steps:
    - name: 소스 코드 가져오기
      uses: actions/checkout@v3

    - name: Docker Hub 로그인
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_USERNAME }}
        password: ${{ secrets.DOCKER_PASSWORD }}

    - name: Docker 빌드 및 푸시
      uses: docker/build-push-action@v4
      with:
        context: .
        file: ./Dockerfile.prod # 아까 만든 배포용 도커파일 사용
        push: true
        tags: ${{ secrets.DOCKER_USERNAME }}/bookshelf-prod:latest # 태그 지정
        build-args: | # 빌드 시 환경 변수 주입
          VITE_SUPABASE_URL=${{ secrets.VITE_SUPABASE_URL }}
          VITE_SUPABASE_ANON_KEY=${{ secrets.VITE_SUPABASE_ANON_KEY }}
```

---

### Step 3: 마법 감상하기 (Git Push)

```bash
git add .
git commit -m "Add GitHub Actions workflow"
git push origin main
```

1.  GitHub 저장소의 **Actions** 탭
2.  **`Docker Build & Push`** 작업이 돌아가고 있음
3.  초록색 체크 표시(`✅`)가 뜨면 성공! 이제 Docker Hub에도 자동으로 새 이미지가 올라와 있을 것
