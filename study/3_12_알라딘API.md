도서 검색 -> 알라딘 API

***[ 1. 주의할 점 : CORS 에러 ]***
- CORS ( Cross-Origin Resource Sharing )
- 프론트엔드 개발에서 흔히 마주치는 에러
- 웹 브라우저(크롬, 사파리 등)는 기본적으로 **"우리집 마당에서 가져온 물건만 안전하다"**고 생각하는 아주 엄격한 보안 규칙을 가지고 있습니다. 이 규칙을 **동일 출처 정책 (SOP, Same-Origin Policy)**이라고 부릅니다.
- 우리가 만든 리액트 페이지에서 알라딘에게 "책 정보 좀 줘!" 하고 요청(fetch)을 보냅니다. 알라딘 서버는 정상적으로 "응, 여깄어!" 하고 데이터를 던져줍니다. 그런데 내 브라우저(크롬)가 데이터를 딱 받아오더니 문 앞에서 검사를 합니다. "어? 너 방금 http://localhost:5173(우리집)에서 보낸 요청인데, 데이터는 http://www.aladin.co.kr(남의 집)에서 왔네? 남의 집 물건은 위험하니까 차단할게!"
- 원래 정석적인 해결법은 '남의 집(알라딘 서버)' 주인이 "어, 저기 localhost:5173에서 오는 요청은 믿을만하니까 허락해 줘!" 하고 쪽지(헤더)를 붙여서 보내주는 것입니다. 하지만 우리가 알라딘 서버를 직접 수정할 수는 없죠? 그래서 사용하는 방법이 **우회 수단(프록시, Proxy)**입니다. 
- **프록시(Proxy)란**? (대리인 세우기) 내 브라우저와 알라딘 서버 사이에 '대리인(Vite 서버)'을 하나 둡니다.

***[ 2. 데이터 형식은 JSON으로 받기 ]***
- 알라딘 API는 XML로 데이터를 우리에게 전달
- 편하게 다루기 위해 쿼리 파라미터에 &output=js(또는 json)를 추가해서 배열과 객체 형태로 받기

***[ 3. 코딩 순서 가이드 ]*** ( in Searchpage.tsx )
- 상태 만들기 (State)
    - 사용자가 검색창에 입력한 단어를 저장할 State (const [keyword, setKeyword] = useState(''))
    - API에서 받아온 도서 목록 배열을 저장할 State (const [searchResults, setSearchResults] = useState([]))
- 검색 함수 만들기
    - <form onSubmit={...}> 부분에 연결할 검색 실행 함수 생성
    - fetch or axios를 사용해 알라딘 API url 호출
    - URL example : /aladin-api/ttb/api/ItemSearch.aspx?ttbkey=본인키&Query=${keyword}&QueryType=Keyword&MaxResults=10&start=1&SearchTarget=Book&output=js&Version=20131101
- 데이터 받아와서 화면에 그리기
    - fetch 완료 후 받아온 데이터(JSON) 안의 .item 배열을 setSearchResult를 통해 저장
    - searchResults.map(...)
