# Nginx 리버스 프록시 (Reverse Proxy) 아키텍처 구축 가이드

## 1. 아키텍처 변경의 목적 (왜 하는건가요?)

### 🚨 기존 방식의 문제점 (포트 개방)
앞서 우리는 `docker-compose.yml`을 통해 **React 앱(5173)**, **Prometheus(9090)**, **Grafana(3000)** 세 가지 컨테이너를 실행했습니다. 
이때 각 컨테이너의 `ports`를 전부 열어두어 브라우저에서 `localhost:5173`, `localhost:3000` 등으로 직접 접속할 수 있게 했습니다.
하지만 실제 서버(운영 환경)에서 이렇게 여러 포트를 동네방네 열어두면 **보안에 매우 취약(해커들의 타겟)**해집니다.

### 🛡️ Nginx 리버스 프록시 도입 (문지기 1명만 세우기)
- **외부로 나가는 문은 오직 하나(80번 포트)만 엽니다.**
- 가장 앞단에 **Nginx(엔진엑스)**라는 문지기 컨테이너를 세웁니다.
- 사용자가 `localhost`로 접속하면 React 앱으로, `localhost/grafana/`로 접속하면 Grafana로 Nginx가 알아서 **내부 교통정리(라우팅)**를 해줍니다.
- 뒤에 있는 앱, 프로메테우스, 그라파나는 **포트를 완전히 닫고 도커 내부 네트워크 안에서만 안전하게 통신**하게 됩니다.

## 2. Nginx 설정 파일 (`nginx-dev.conf`) 한 줄씩 파헤치기

Nginx 설정 파일은 "외부에서 들어온 요청을 내부의 어떤 컨테이너로 토스(Routing)할 것인가"를 정의하는 지도입니다.

```nginx
server {
    # 외부에서 오는 모든 요청을 80번 포트(기본 HTTP 포트)에서 대기합니다.
    listen 80; 

    # ----------------------------------------------------
    # [1] 프론트엔드 라우팅 (경로: /)
    # ----------------------------------------------------
    # 사용자가 도메인 뒤에 아무것도 안 붙이고 접속하면(localhost/) 이 블록이 실행됩니다.
    location / {
        # 도커 네트워크 안쪽에 있는 bookshelf-app 컨테이너의 5173 포트로 요청을 넘깁니다.
        proxy_pass http://bookshelf-app:5173/;
        
        # 원래 사용자가 요청했던 도메인 호스트 이름을 그대로 넘겨줍니다.
        proxy_set_header Host $host;
        # 사용자의 진짜 IP 주소를 숨기지 않고 앱에 전달합니다. (접속 로그 기록 등 목적)
        proxy_set_header X-Real-IP $remote_addr;
        
        # [Vite 특화 설정] 
        # React 개발 환경(Vite)은 코드를 저장하면 화면이 바로 새로고침(HMR) 되는 기능이 있습니다.
        # 이 기능은 '웹소켓'이라는 실시간 통신을 쓰기 때문에 Nginx가 웹소켓을 허용하도록 업그레이드 해주는 설정입니다.
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # ----------------------------------------------------
    # [2] 그라파나 라우팅 (경로: /grafana/)
    # ----------------------------------------------------
    # 사용자가 localhost/grafana/ 로 접속하면 이 블록이 실행됩니다.
    location /grafana/ {
        # 도커 안쪽에 숨어있는 grafana 컨테이너의 3000 포트로 몰래 넘겨줍니다. 
        proxy_pass http://grafana:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # ----------------------------------------------------
    # [3] 프로메테우스 라우팅 (경로: /prometheus/)
    # ----------------------------------------------------
    location /prometheus/ {
        # 도커 안쪽에 숨어있는 prometheus 컨테이너의 9090 포트로 넘겨줍니다.
        proxy_pass http://prometheus:9090/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 3. `docker-compose.yml` 한 줄씩 파헤치기

단순히 Nginx 설정만 바꾼다고 되는 게 아니라, 뒤에 숨은 애들도 자기가 뒤에 숨어있고 '하위 경로'로 접속된다는 사실을 알아야 합니다.

```yaml
services:
  # ✨ [가장 앞단] 문지기 역할을 할 Nginx 컨테이너
  nginx-proxy:
    image: nginx:stable-alpine
    ports:
      - "80:80" # [핵심] 전체 서비스 중 유일하게 바깥 세상(내 맥북)과 통하는 구멍입니다.
    volumes:
      # 아까 만든 nginx-dev.conf 파일을 Nginx 컨테이너의 핵심 설정 파일 공간에 덮어씌웁니다.
      - ./nginx-dev.conf:/etc/nginx/conf.d/default.conf:ro
    depends_on:
      # Nginx가 실행되기 전에 반드시 아래 3개 서비스가 먼저 켜져 있어야 한다는 순서 보장 규칙입니다.
      - bookshelf-app
      - prometheus
      - grafana

  # ----------------------------------------------------
  bookshelf-app:
    build: .
    # [핵심] 기존에 있던 ports: - "5173:5173" 가 삭제되었습니다.
    # 이제 바깥(맥북이나 크롬)에서는 5173으로 절대 다이렉트 접속이 불가능합니다. 오직 Nginx만 이 컨테이너와 대화할 수 있습니다.
    ...

  # ----------------------------------------------------
  prometheus:
    image: prom/prometheus:latest
    # [핵심] ports 삭제됨. 
    ...
    # [하위 경로 꼼수] 프로메테우스는 기본적으로 자기가 '/' (최상단) 경로에 있다고 생각합니다.
    # 하지만 우리는 브라우저에서 '/prometheus/' 로 접속할 것이기 때문에, 프로메테우스 실행 옵션에 
    # "야 너는 이제부터 /prometheus 라는 하위 경로에서 뼈를 묻어라!" 라고 강제 주입하는 명령어입니다.
    command:
      - "--config.file=/etc/prometheus/prometheus.yml"
      - "--web.external-url=/prometheus/"

  # ----------------------------------------------------
  grafana:
    image: grafana/grafana:latest
    # [핵심] ports 삭제됨.
    ...
    # [하위 경로 꼼수 2] 그라파나도 자기가 서버의 주인('/')인 줄 압니다.
    # 환경 변수(environment)를 주입해서, "너의 진짜 주소는 localhost/grafana/ 니까, 
    # 모든 이미지나 CSS 파일을 찾을 때 /grafana/ 아래에서 찾아라!" 라고 훈련시키는 설정입니다.
    environment:
      - GF_SERVER_DOMAIN=localhost
      - GF_SERVER_ROOT_URL=%(protocol)s://%(domain)s/grafana/
      - GF_SERVER_SERVE_FROM_SUB_PATH=true
```

## 3. 실행 및 접속 테스트 방법

모든 코드가 준비되었으니 터미널에서 변경된 설계도(`docker-compose.yml`)를 기반으로 오케스트라를 새로 띄워봅니다.

### 🚀 명령어 실행
```bash
# 최상위 bookshelf 폴더에서 
docker-compose up -d --build --remove-orphans
```

### 🎯 접속 확인 (포트 번호 없이 접속!)

이제 뒤에 지저분한 `:5173` 이나 `:3000` 같은 숫자를 붙이지 않아도 됩니다!

1. **프론트엔드 (React)**
   👉 [http://localhost](http://localhost) 
2. **그라파나 대시보드 (Grafana)**
   👉 [http://localhost/grafana/](http://localhost/grafana/) 
3. **프로메테우스 (Prometheus)**
   👉 [http://localhost/prometheus/](http://localhost/prometheus/)

> **💡 데브옵스 전문가로서의 한걸음**
이렇게 뒷단 서버들의 엉덩이(포트)를 꽁꽁 숨기고 80번 포트 앞문 단 하나만 열어두는 **설계(아키텍처)의 미학**을 방금 완벽하게 구축하셨습니다. 터미널 명령어를 입력하시고 새롭게 바뀐 쾌적한 주소로 접속해 보세요!
