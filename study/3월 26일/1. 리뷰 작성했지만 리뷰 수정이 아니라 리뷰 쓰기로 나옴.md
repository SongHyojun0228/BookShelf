## 문제 상황 
> 리뷰 작성을 완료하고 모달이 닫혔을 때, 부모 컴포넌트(예: 내 서재 페이지)가 "리뷰가 새로 작성(또는 수정)되었다"는 사실을 모르고 이전 데이터를 그대로 보여주고 있기 때문에 발생하는 현상

## 방법 1 : 부모 컴포넌트에서 "새로고침 함수"를 넘겨주기 
- 부모 컴포넌트(MyShelf)에서 데이터를 다시 불러오는 함수 생성
- 부모 컴포넌트에서 <Reviewmoal>을 호출할 때 props로 그 함수를 넘겨 주기
- Reviewmodal.tsx : insert나 update가 성공적으로 완료되어 alert을 띄운 직후, props로 받은 onSuccess()를 실행

## 방법 2 : React Query(or SWR) 데이터 무효화 사용
- 만약 프로젝트에서 React Query 같은 서버 상태 관리 라이브러리를 쓰고 계신다면 아주 쉽게 해결
- 리뷰 쓰기(mutation)가 성공했을 때, queryClient.invalidateQueries(['books']) 처럼 캐싱된 데이터를 무효화시키면 알아서 최신 데이터를 다시 가져와 화면을 업데이트
