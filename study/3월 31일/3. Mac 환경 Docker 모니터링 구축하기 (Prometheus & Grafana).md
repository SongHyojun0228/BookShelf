# Mac 환경 Docker 완벽 모니터링 구축기 (Prometheus & Grafana)

도커(Docker)로 만든 서비스들이 잘 돌아가고 있는지(CPU, 메모리, 동작 중인 컨테이너 수 등)를 실시간으로 감시하기 위해 **모니터링 시스템**을 구축했습니다. 특히 Mac(Apple Silicon M1/M2/M3) 환경에서는 일반적인 방식이 통하지 않아 겪었던 시행착오와 가장 완벽한 해결책을 정리합니다.

---

## 1. 구성 요소의 역할 (이것들이 뭔가요?)

### 🐳 도커 엔진 메트릭 (소리꾼 / 정보 제공자)
*   **원래 하려던 것 (cAdvisor)**: 원래는 도커 컨테이너마다 찰싹 붙어서 상태를 읽어오는 구글의 `cAdvisor`라는 툴을 쓰려고 했습니다.
*   **문제 발생**: Mac(맥북) 아키텍처는 리눅스와 달라서, 도커가 가상화(VM) 안에서 돌아갑니다. 이 때문에 cAdvisor가 컨테이너들의 '이름(Name)'표기를 제대로 읽어오지 못하고 `ID`만 대충 던져주는 고질병이 있습니다.
*   **해결책**: cAdvisor를 과감히 버리고, **Docker Desktop 프로그램 자체(Docker Engine)**가 직접 뿜어내는 공인된 메트릭(상태 데이터)을 사용하기로 수정했습니다.

### 🗄️ Prometheus (프로메테우스 - 데이터 수집가)
*   프로메테우스는 **시계열 데이터베이스(TSDB)**입니다.
*   "5초마다 도커 엔진한테 가서 CPU는 몇 개 쓰는지, 컨테이너는 몇 개 켜져 있는지 물어보고 창고에 차곡차곡 쌓아둬!" 라는 역할을 수행합니다. (`prometheus.yml` 파일이 어디서 정보를 가져올지 알려주는 지도입니다.)

### 🎨 Grafana (그라파나 - 시각화 예술가)
*   그라파나는 창고(프로메테우스)에 쌓여있는 숫자들을 사람이 보기 편하게 **그래프나 화려한 대시보드로 색칠해 주는 도화지**입니다.
*   데이터를 모으는 능력은 없지만, 가져온 데이터(Prometheus Data Source)를 기반으로 예쁜 차트를 자유롭게 그릴 수 있습니다.

---

## 2. 실행 및 구축 방법 (How to Run)

### Step 1. 도커 데스크탑 메트릭 활성화 (Mac 전용 설정 추가)
도커 데스크탑이 자신의 상태(Metrics)를 밖으로 알려주도록 허락해 주어야 합니다.

1.  Mac 위쪽 메뉴바에서 **Docker 아이콘 🐳 -> [Settings]** 를 누릅니다.
2.  왼쪽 메뉴에서 **[Docker Engine]** 탭에 들어갑니다.
3.  아래의 코드를 json 파일 안에 추가합니다.
    ```json
    {
      "experimental": true,
      "metrics-addr": "0.0.0.0:9323"
    }
    ```
4.  우측 하단의 **[Apply & restart]** 를 눌러 도커를 완전히 재시작합니다.

### Step 2. `prometheus.yml` 작성 (어디서 수집할지 지도 그리기)
프로젝트 폴더(최상단)에 아래 내용으로 문서를 만듭니다. (도커 데스크탑 내부 주소인 `host.docker.internal:9323`를 가리켜야 합니다!)

```yaml
global:
  scrape_interval: 5s # 5초마다 데이터 수집

scrape_configs:
  - job_name: "docker" # ⭐️ 남이 만든 대시보드들은 보통 이 이름을 씁니다.
    static_configs:
      - targets: ["host.docker.internal:9323"] # 도커 데스크탑 직접 연결
```

### Step 3. `docker-compose.yml` 실행!
앱과 함께 프로메테우스, 그라파나를 한 번에 실행시킵니다.

```yaml
services:
  # 내 앱
  bookshelf-app:
    build: .
    ports:
      - "5173:5173"

  # 데이터 수집가 (Prometheus)
  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml:ro

  # 시각화 전문가 (Grafana)
  grafana:
    image: grafana/grafana:latest
    container_name: grafana
    ports:
      - "3000:3000"
```

터미널을 켜고 다음 명령어를 칩니다.
```bash
docker-compose up -d --build
```

### Step 4. 그라파나 대시보드 연결 (최종 시각화)
1.  브라우저에서 **[localhost:3000](http://localhost:3000)** 에 접속합니다. (초기 아이디/비밀번호는 `admin/admin`입니다.)
2.  **왼쪽 메뉴 -> [Connections] -> [Data sources]** 에서 `Prometheus`를 선택하고, HTTP URL 칸에 **`http://prometheus:9090`** 을 적은 뒤 Save를 누릅니다.
3.  **[Dashboards] -> [New] -> [Import]** 로 들어가서 남이 잘 만들어둔 템플릿의 고유 번호(ID)를 적거나(예: 1229), 또는 우리가 직접 만든 완벽한 커스텀 `.json` 파일을 업로드(Upload) 하면 끝입니다!

## 💡 회고 및 트러블슈팅 (왜 그토록 N/A 에러가 떴는가)
그라파나에 외부 템플릿(ID: 1229 등)을 불러왔을 때 **N/A**나 **No Data**가 줄기차게 떴던 두 가지 핵심 원인은 다음과 같습니다.

1.  **변수(`job_name`)의 불일치**: 1229 템플릿은 프로메테우스의 `job_name`이 정확히 "docker"라고 가정하고 하드코딩 되어 있습니다. 우리가 초반에 "docker-containers"라고 이름을 지었기 때문에 서로 정보를 찾지 못했습니다.
2.  **그라파나 변수($instance) 버그**: 다른 사람이 만든 템플릿의 정규식(Regex) 변수가 현재 내 시스템 환경에 완벽하게 일치하지 않아 값이 비어버리면, 연결된 모든 쿼리가 연쇄적으로 깨지며 No Data가 뜹니다.

때문에 남의 대시보드를 억지로 쓰며 고통받는 것보다, 프로메테우스가 가져온 생생한 데이터(`engine_daemon_engine_cpus_cpus` 등)를 **우리가 패널에 직접 하드코딩해서(Custom Direct)** 넣는 것이 Mac 로컬 환경 구축 연습에서는 가장 스트레스 없고 확실한 방법이었습니다.
