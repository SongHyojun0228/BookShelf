# 여러 개의 useEffect로 나누어야 하는 3가지 상황

> 리액트에서 하나의 큰 useEffect 안에다가 모든 로직을 때려넣지 않고, 여러 개로 쪼개야하는 상황이 있다.

---

## 1. 연쇄적인 순서(실행 타이밍?) 보장이 필요할 때 = 비동기 체인
### A 작업이 끝나고, 그 결과로 B 작업을 시행할 수 있을 때? useEffect를 나눠야한다.
- 왜??? React의 useState 상태 업데이트는 비동기적으로(렌더링이 끝나야) 반영되기 때문!
- 잘못된 예 ( 하나로 묶었을 때 )
```typescript
useEffect(() => {
  // 1. 책 검색해서 상태를 업데이트
  const data = fetchBook(id);
  setBookData(data); 

  // 2. 방금 세팅한 데이터로 리뷰 검색 (❗️에러 발생)
  fetchReviews(bookData.title); 
  // ⚠️ 위 코드가 실행되는 시점에 bookData는 여전히 초기값(null)입니다! 
  // 상태 업데이트(setBookData)는 다음 렌더링 때 반영되기 때문입니다.
}, [id])
```

- 올바른 예 ( 나눴을 때 )```typescript
```typescript
useEffect(() => { fetchBook(id) }, [id]);  // 1. id가 있으면 책 조회
useEffect(() => { fetchReviews(bookData.title) }, [bookData]); // 2. bookData가 들어오면 리뷰 조회
}, [id])
```

## 2. 의존성이 서로 다를 때 
### 서로 관련 없는 로직들이 똑같은 타이밍에 실행되는 것을 막기 위함
- 잘못된 예
```typescript
useEffect(() => {
  startTimer();       // 시간에 관련된 로직
  search(keyword);    // 검색에 관련된 로직
}, [time, keyword]); 
```
- 한 글자 칠 때마다 상관도 없는 타이머가 재시작되고, 타이미가 1초가 지날 때마다 상관없는 검색이 실행.

- 올바른 예
```typescript
useEffect(() => { startTimer() }, [time]); // 시간에만 반응
useEffect(() => { search(keyword) }, [keyword]); // 검색에만 반응
```
